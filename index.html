Abstract

Sampling-based motion planning algorithms are widely used for motion planning of robotic manipulators, but they often struggle with sample inefficiency in high-dimensional configuration spaces due to their reliance on uniform or hand-crafted informed sampling primitives. Neural informed samplers address this limitation by learning the sampling distribution from prior planning experience to guide the motion planner towards planning goal. However, existing approaches often struggle to encode the spatial structure inherent in motion planning problems. To address this limitation, we introduce Graph-based Attention Masking for Spatial- and Embodiment-aware Motion Planning (GAIDE), a neural informed sampler that leverages both the spatial structure of the planning problem and the robotic manipulator's embodiment to guide the planning algorithm. GAIDE represents these structures as a graph and integrates it into a transformer-based neural sampler through attention masking. We evaluate GAIDE against baseline state-of-the-art sampling-based planners using uniform sampling, hand-crafted informed sampling, and neural informed sampling primitives. Evaluation results demonstrate that GAIDE improves planning efficiency and success rate.

Key Contributions

Graph Construction

A unified graph encodes both the manipulator's kinematic chain (embodiment graph) and the spatial relationships between the robot and workspace obstacles (spatial graph).

Attention Masking

The graph adjacency matrix is used as a binary attention mask inside the transformer encoder, constraining information flow to structurally meaningful token pairs.

Superior Path Quality

GAIDE consistently achieves the lowest planning cost across all six benchmark tasks, outperforming all sampling-based baselines including neural informed samplers.

Method

GAIDE is a neural informed sampler embedded within a bidirectional sampling-based planner. At each planning step it receives the current joint configuration, the goal configuration, the robot point cloud, and the workspace point cloud, and outputs a delta joint angle that extends the planning tree toward the goal. The framework is built around two core ideas: constructing a scene graph that captures planning structure, and wiring that graph directly into the transformer via attention masking.

Graph Construction

GAIDE builds a unified graph over two complementary representations of the planning scene:

  • Embodiment graph. An undirected graph is constructed over the downsampled robot point cloud. Each downsampled point is a node, and edges follow the manipulator's kinematic chain, connecting every link to its adjacent neighbors. This implicitly encodes the robot's kinematic topology without explicit kinematic parameters.
  • Spatial graph. A directed, fully connected graph connects every node in the downsampled workspace point cloud to all robot graph nodes. This captures the spatial relationship between each obstacle and every part of the manipulator body, providing the planner with workspace-aware context.

Both point clouds are downsampled using PointNet++ set abstraction layers. The resulting combined adjacency matrix is used directly as an attention mask inside the transformer encoder.

Figure 1 — Graph Construction. (Top) The robot point cloud is downsampled via PointNet++ to produce robot nodes connected along the kinematic chain. (Bottom) The workspace point cloud is similarly downsampled to workspace nodes. A directed graph then connects every workspace node to all robot nodes, forming the spatial adjacency matrix used for attention masking.

GAIDE Architecture

Given the graph adjacency matrix A, GAIDE processes four input streams through dedicated encoders: a shared MLP embeds the current and goal joint configurations; two PointNet++ set abstraction stacks encode the robot and workspace point clouds. Robot, goal, and current-step embeddings are fused and passed as a token sequence to the transformer encoder.

The encoder interleaves masked attention layers — where off-graph token pairs are suppressed by setting their attention logits to −∞ — with standard vanilla attention layers that allow unconstrained global aggregation. This interleaving preserves long-range dependencies while enforcing structural inductive biases, avoiding the oversmoothing that plagues deep GNN message-passing. Dropout is applied at inference time to introduce stochasticity, which is essential for probabilistic completeness of the planning algorithm. A lightweight transformer decoder takes a single learnable token and, conditioned on the encoder output, predicts the delta joint angle δqt.

Figure 2 — GAIDE Network Architecture. Current and goal configurations are encoded by a shared MLP; robot and workspace point clouds are encoded by PointNet++ set abstraction layers. The transformer encoder alternates masked attention (constrained by the graph adjacency matrix) and vanilla attention layers. The transformer decoder predicts a delta joint angle to grow the planning tree.

Results

We evaluate GAIDE on six held-out planning tasks against three classes of baselines: uniform samplers (Bi-RRT, RRT*), heuristic-based informed samplers (IRRT*, BIT*), and neural informed samplers (MPNets, SIMPNet). All planners share the same bidirectional planning framework and PyBullet collision checker. Classical planners are given a time budget equal to GAIDE's average planning time per task. Three metrics are reported: planning time T [s], success rate S [%], and planning cost C (path length in configuration space).

Planning Cost

Planning cost measures the path length in configuration space (lower is better). GAIDE achieves the lowest average cost of 4.81 ± 1.63 across all tasks — substantially below Bi-RRT (16.2 ± 7.05), RRT* (6.98 ± 6.47), IRRT* (10.9 ± 6.98), and BIT* (9.0 ± 5.17), confirming that graph-structured attention produces shorter, higher-quality paths compared to both classical and neural baselines.

Ablation Study

We ablate the attention masking design with two controlled variants to isolate the contribution of the interleaved masking strategy:

  • GAIDE-V (Vanilla Transformer). Standard encoder-decoder transformer with no attention masking — processes all tokens uniformly without encoding spatial or kinematic structure.
  • GAIDE-H (Hard masking). Applies the graph attention mask at every encoder layer rather than interleaving. This prevents the decoder from fully attending to workspace embeddings, causing spatial information to be masked out at each layer.

GAIDE's interleaved design — alternating masked and vanilla layers — consistently outperforms both variants, confirming that the balance between structured and global attention is critical.

Real-World Deployment

We deploy GAIDE on a physical UR5e robotic arm without any additional training or fine-tuning. The scene is represented using point cloud data from a calibrated Intel RealSense D435i RGB-D camera, with the camera-to-robot-base transform estimated via AprilTag markers. Collision checking uses a spherical approximation of the robot geometry. Three demonstrations are shown: a bin task (Demo I) and two shelf-insertion tasks of increasing difficulty (Demo II, Demo III). Each demo shows the physical robot (top) alongside the corresponding simulated point cloud viewed by the planner (bottom).

Figure 6 — Real-World Deployment. Demo I: bin task. Demo II & III: shelf-insertion tasks with increasing obstacle complexity. Top rows show the physical UR5e robot; bottom rows show the simulated point cloud scene used by the planner. GAIDE plans collision-free paths in all three scenarios without retraining.
